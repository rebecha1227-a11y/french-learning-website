# Français Quest · TCF Canada 备考驾驶舱

一个专为 TCF Canada 备考打造的个人学习管理工具，基于 Édito A1/A2 教材主线，整合任务管理、练习判题、错题复盘与学习追踪。纯 HTML + CSS + JS 实现，Supabase 云端同步可选。

---

## 如何打开

```
# 方式一：直接双击（推荐）
双击项目根目录的 index.html

# 方式二：本地服务器（避免 file:// 跨域限制）
npx serve .
# 然后访问 http://localhost:3000
```

## 各模块使用方法

### 首页仪表板

打开网站默认进入首页，分三栏展示：

**左栏 — 用户信息与导航**
- 用户卡片显示学习统计：累计完成任务数、学习连续性%、计划执行率%、连续打卡天数、今日学习日志（完成/总数）
- 点击右上角「登录」按钮可注册/登录 Supabase 账号，登录后数据云端同步；未登录为本地访客模式，数据存储在浏览器 localStorage
- 「功能导览」网格可快速跳转到各模块（与顶部导航栏等价）

**中栏 — 今日任务与课程进度**
- 今日任务卡：可在首页直接添加任务（输入标题 → 选类型 → 点「+」），勾选即完成
- 课程进度卡：显示前 8 个单元的完成进度条，点「全部」跳转课程主线页

**右栏 — 数据与建议**
- 倒计时：显示距 TCF 考试剩余天数（默认 2027-05-01，可自定义）。点击日期输入框修改考试日期，点「保存」生效
- 学习热力图：展示近 7 周（日历视图）每日学习完成情况，颜色越深代表当日完成任务越多
- AI 备考建议：显示 3 条固定建议（当前版本为静态内容，未来接入动态推荐）

---

### 今日学习（任务管理）

**任务看板**

页面顶部三格指标：今日完成数/总数、近7天完成任务数、考试倒计时天数。

四个 Tab 切换时间维度：
- **今日**：仅看今天（默认）。日期选择器可跳转到任意日期查看/编辑当天任务
- **本周**：当前自然周内所有任务，按日期分组
- **本月**：当前自然月所有任务
- **今年**：当前年度所有任务

**添加任务**
1. 在输入框填写任务标题
2. 点击右侧下拉框选择任务类型（讲义 / 练习 / 复盘 / 真题 / 视频 / 复习）
3. 点「+」按钮添加，任务出现在当日列表

**管理单条任务**
- 勾选复选框 → 标记完成（记录完成时间戳，自动更新学习日志）
- 点右侧「···」设置菜单 → 可选「顺延」（移到明天/后天/自定义日期）或「删除」
- 页面右上角「显示已完成(N)」复选框：控制已完成任务是否显示（已完成任务默认沉底）

**自动顺延**
- 右上角「设置」按钮 → 开启「自动顺延」后，每天首次打开网站时，系统自动把昨天未完成的任务顺延到今天
- 开启时可设置是否显示顺延通知

**补卡**
- 在任务设置菜单中选「补卡」→ 选择历史日期 → 将任务标记为该日期完成（同步写入学习日志）

### 课程主线

页面展示 Édito A1（10 个单元）和 Édito A2（若干单元）的卡片列表。

每张单元卡片显示：单元编号、标题、学习焦点描述、完成进度条（%）

**进入单元详情**：点击任意单元卡片

---

### 单元详情（讲义 + 练习）

**顶部操作区**
- 「← 返回课程」：回到课程主线列表
- 「↓ 讲义」「↓ 练习」：分别导出当前单元的讲义/练习为 PDF（调用浏览器打印）
- 进度条区域：显示讲义完成状态 + 练习正确率 + 单元综合进度%
- 「标记讲义已学完」按钮：手动标记讲义已读，更新单元进度

**📖 讲义 Tab**

结构化展示该单元内容：
- 单元概述（Unité 简介）
- 学完你应该会（Can-do 列表）
- 语法精讲：包含语法说明、变位表/对比表、例句、常见错误陷阱
- 词汇组：按主题分组，每词含法语原文 + 中文释义 + 例句
- 语音重点（phonétique 要点）
- TCF 应试提示

**✏️ 练习题 Tab**

按题型分组（如「tu 或 vous？」「词形匹配」等），每题包含：
- 题目 prompt（可带上下文句子）
- 输入框：填写你的答案
- 「提交」按钮：系统判题
  - ✅ 正确：绿色反馈，练习统计 +1
  - ❌ 错误：红色反馈显示正确答案，**自动将本题加入错题库**（去重）
- 「▶ 查看答案」展开：直接查看标准答案和解析，不计入判题统计

**判题规则**
- 支持多答案（用 `/` 或 `;` 分隔的标准答案中任一均算对）
- 忽略大小写、标点、法语变音符（e/é/è/ê 视为等价）
- 相似度容错：答案与标准答案相似度 ≥ 85% 也算对

---

### 错题复盘

**新增错题（手动录入）**

表单字段：
- 日期（默认今天）
- 来源（下拉选择：课程练习/TCF 真题/听力/阅读等）
- 题目内容
- 我的错误答案
- 正确答案
- 错误原因 & 思考（文本域）
- 反思（下次如何做）
- 标签：多选（语法 / 词汇 / 听力 / 阅读 / 审题 / 表达）+ 可自定义新标签

点「保存错题」入库，点「清空」重置表单。

> **注意**：在单元练习页答错题会自动进入错题库，无需手动录入。手动录入适合记录 TCF 真题、听力等非系统内练习的错误。

**错题列表**（表单下方）

表格展示所有错题，列：日期 / 来源 / 题目 / 我的答案 / 正确答案 / 错因 / 反思 / 标签 / 操作

- 错因和反思列支持**内联编辑**：直接点击单元格修改，自动保存
- 标签列：点击可多选/取消标签，管理该题的分类
- 操作列：「重练」按钮 → 弹窗展示题目，重新作答；「掌握」按钮 → 标记为已掌握（灰化显示）
- 顶部标签筛选器：按标签过滤显示特定类型错题
- 「导出 CSV」：将错题表导出为 CSV 文件

### 账号与云同步

**注册 / 登录**
1. 点击首页用户卡片右上角「登录」按钮
2. 选择「注册」输入邮箱 + 密码，或直接「登录」已有账号
3. 登录后用户卡片显示「已登录」，数据自动与 Supabase 云端同步

**数据隔离**
- 访客模式（未登录）：数据仅存本地 localStorage，换浏览器/设备即丢失
- 登录模式：数据存储在云端，支持多设备同步

**手动同步**（登录后可用）
- 用户卡片内「从云端拉取」：将云端数据覆盖到本地（本地改动会丢失，操作前确认）
- 「推送到云端」：将本地数据上传覆盖云端

**首次登录同步决策**
登录时若本地和云端都有数据，系统会弹出三选一对话框：
1. 保留云端数据（覆盖本地）
2. 保留本地数据（推送到云端）
3. 暂不处理（保持本地数据，不同步）

## 数据存储说明

所有数据本地存储在浏览器 `localStorage`，登录后自动同步到 Supabase。

| 数据类型 | 说明 |
|---------|------|
| 任务列表 | 所有学习任务、完成状态、顺延历史 |
| 错题库 | 所有错题记录、标签、复习状态 |
| 学习日志 | 每日完成/总数，用于热力图和连续打卡统计 |
| 练习统计 | 各单元答题总数和正确数，用于计算正确率 |
| 单元学习状态 | 讲义是否标记完成、已做练习题列表 |
| 考试日期 | 用户设置的 TCF 考试日期（默认 2027-05-01） |
| 自动顺延设置 | 是否开启、上次执行日期 |
| 错题标签库 | 用户自定义的标签列表 |

清空浏览器数据会导致**本地数据永久丢失**（已登录用户可从云端重新拉取）。

## UX 问题清单（用户视角 + 产品经理视角）

> 以下问题按严重程度排列，供后续迭代参考。

---

### 🔴 严重：影响核心流程

**1. 题库中心和视频补充是「假入口」**
- 问题：顶部导航里有「题库中心」和「视频补充」，点进去是占位文案或无功能的输入框，用户会以为网站坏了。
- 建议：在导航项上加「(建设中)」标注，或直接从顶部导航隐藏这两个入口，只保留首页功能导览中的入口（加灰显示）。

**2. 首页中栏和「今日学习」内页功能重叠导致困惑**
- 问题：首页中栏「今日学习任务」已经可以添加/勾选任务；点「今日学习」进入内页，还能做同样的事。两个入口产生的数据是同一份，但用户不明白为什么要有两个地方。
- 建议：明确分工——首页中栏只做「今日任务一览」（只读或快速勾选），完整的任务管理操作（顺延、删除、多维度查看）收归「今日学习」内页。

**3. 错题自动入库 vs 手动录入逻辑不清**
- 问题：练习答错自动入错题库，但「错题复盘」页一进去就是一个手动填写表单，完全没有提示自动入库已经发生。新用户会以为每次答错都需要手动填一遍，极大地增加认知负担，甚至可能跳过练习。
- 建议：「错题复盘」页默认展示错题列表，手动新增表单放次要位置（折叠或弹窗）；在练习答错弹窗里加一句「已自动入库」的提示。

---

### 🟠 中等：明显影响体验

**4. 单元进度全是 0%，新用户没有起点引导**
- 问题：课程主线所有单元进度都是 0%，没有任何「从这里开始」的引导。新用户不知道该点哪个单元，也不知道完成之后进度会如何更新。
- 建议：给第一个单元加「推荐起点」标签，进度为 0 时显示引导提示「点击单元开始学习」。

**5. AI 建议是固定文案，名实不符**
- 问题：「AI 备考建议」显示的 3 条建议是写死在代码里的静态文字，和用户的实际学习状态（哪个单元在学、错题分布、上次学习时间）完全无关。叫「AI 建议」会让用户预期过高。
- 建议：要么去掉「AI」这个词，改为「每日学习提示」；要么基于用户的真实数据（近期错题类型、未完成单元、连续天数）生成动态提示。

**6. 自动顺延默认关闭且入口隐蔽**
- 问题：自动顺延是一个对备考者非常有价值的功能，但默认关闭，且入口只有一个小「设置」按钮（右上角，不显眼）。大多数用户不会主动去找，就永远不知道这个功能存在。
- 建议：首次进入「今日学习」时弹一次引导（「是否开启自动顺延？未完成任务将在次日自动延续」），让用户主动决策。

**7. 「380 天」倒计时芯片缺乏说明**
- 问题：顶部导航右侧显示「380 天」，新用户完全不知道这是什么的倒计时（考试？课程？）。鼠标悬停也没有 tooltip。
- 建议：改为「380 天 · 距 TCF」或加 hover tooltip 说明「距 TCF Canada 考试还有 380 天」。

**8. 讲义/练习 Tab 切换和导出按钮布局混淆**
- 问题：单元详情页的左上区域有「📖 讲义」「✏️ 练习题」两个 Tab，同时右上角又有「↓ 讲义」「↓ 练习」两个下载按钮。用户初次看到会搞不清「↓ 练习」是切换到练习 Tab 还是下载练习。
- 建议：下载按钮改为明确的图标+文字（如「⬇ 导出讲义 PDF」），并与 Tab 组拉开视觉距离。

**9. 练习答题后缺乏成就感反馈**
- 问题：答对后只有绿色文字「✓ 正确」，无进度积累感；完成一整个练习组后没有「恭喜完成」的小结反馈（共 N 题，正确 M 题，正确率 X%）。
- 建议：每组题做完后展示小结卡片，强化正向反馈闭环。

---

### 🟡 轻微：体验细节

**10. 原生日期选择器（date input）与整体风格割裂**
- 问题：「今日学习」页的日期选择器和「错题复盘」页的日期字段使用浏览器原生 `<input type="date">`，视觉风格（系统默认样式）和整体复古像素风格完全不搭。
- 建议：自定义日期选择器样式，或在 input 上套一层像素风格的 wrapper。

**11. 功能导览和顶部导航完全重复**
- 问题：首页左栏「功能导览」有 5 个大按钮（今日学习/课程主线/题库中心/错题复盘/视频补充），顶部导航栏也有完全相同的 5 个入口。两组导航指向同一目的地，占用了大量首屏空间。
- 建议：首页功能导览改为只展示各模块的关键数据摘要（如「错题库 N 题待复习」「本周完成 N 个单元」），点击进入；而不是重复提供与顶栏一样的跳转按钮。

**12. 已完成任务和未完成任务的视觉区分度不足**
- 问题：已完成任务有删除线，但在多任务列表中视觉上不够醒目，已完成和未完成在一起显示时，未完成任务不够突出。
- 建议：已完成任务降低整体亮度（opacity）并推到列表底部（已实现沉底），同时考虑给未完成的紧急任务加颜色标记。

**13. 「← 首页」返回按钮语义不一致**
- 问题：所有内页左上角都有「← 首页」按钮，但用户从「课程主线」进入「单元详情」时，右上角另有「← 返回课程」，存在两个层级的返回，容易让人困惑「首页」和「返回」的区别。
- 建议：统一为面包屑导航：「⌂ 首页 / 课程主线 / Unité 0」，点哪段就跳到哪里。

**14. 未登录状态无明确利益点说明**
- 问题：用户卡片显示「未登录，本地模式」，但没有任何说明为什么要登录（云同步多设备？防数据丢失？）。登录按钮不突出，用户没有动力去登录。
- 建议：在「登录」按钮旁加一行小字：「登录后数据云端同步，换设备不丢失」。

**15. 移动端完全不可用**
- 问题：整个网站布局基于三栏固定像素宽度，在 720px 以下屏幕会发生布局错位。备考学生很可能在手机上查阅。
- 建议：至少为移动端（<720px）做单栏降级布局，确保核心任务管理流程可用。

## 技术栈

| 层 | 技术 |
|---|---|
| 前端 | 纯 HTML5 + CSS3 + Vanilla JS（无框架） |
| 数据存储 | `localStorage`（本地） + Supabase（云端，登录后） |
| 认证 | Supabase Auth（邮箱密码 + 忘记密码） |
| 字体 | VT323（像素数字）+ Noto Sans SC（正文） |
| 部署 | 任意静态托管（直接双击 index.html 即可） |
