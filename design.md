# Français Quest UI 设计规范（AI Coding 参考版）

## 1. 目标与风格定位
- 网站定位：面向 TCF Canada 备考者的学习驾驶舱（课程主线 + 题库 + 错题 + 进度追踪）。
- 视觉方向：复古像素仪表盘（retro pixel dashboard）+ 学习管理系统。
- 氛围关键词：结构清晰、轻游戏化、信息密度高、可持续盯屏（护眼）。
- 参考源：`french-study-pixel.html` 的三栏布局、像素边框、模块化卡片、数据看板逻辑。

## 2. 本次改色要求（已覆盖）
- 主背景改为复古淡黄色，降低视觉疲劳，适合长时间阅读。
- 保留“像素复古”的骨架和组件语言，不做极简扁平化。

## 3. 设计令牌（Design Tokens）
请优先使用 CSS 变量，不要写死颜色。

```css
:root {
  --bg-deep:      #efe4c8; /* 页面底色：复古淡黄 */
  --bg-panel:     #f6edd9; /* 面板底色 */
  --bg-card:      #f2e7d0; /* 卡片底色1 */
  --bg-card2:     #eadfc6; /* 卡片底色2 */

  --border:       #b8a786; /* 主边框 */
  --border-light: #c9b99a; /* 高亮边框 */

  --accent-blue:  #4f6fa3; /* 信息/主标题 */
  --accent-teal:  #4f8f87; /* 进度/完成 */
  --accent-gold:  #c7923a; /* 重点数字/提醒 */
  --accent-green: #5f8f54; /* 成功状态 */
  --accent-rose:  #b86a6a; /* 风险/错题 */
  --accent-purple:#7a6aa8; /* AI/视频/次强调 */

  --text-main:    #2d261b; /* 主文本 */
  --text-sub:     #5f543f; /* 次文本 */
  --text-dim:     #8a7a5f; /* 弱化文本 */

  --pixel-shadow: 3px 3px 0 #c7b897;
  --pixel-border: 2px solid var(--border);

  --scanline: repeating-linear-gradient(
    0deg,
    rgba(255,255,255,0.06),
    rgba(255,255,255,0.06) 2px,
    rgba(0,0,0,0.02) 2px,
    rgba(0,0,0,0.02) 4px
  );
}
```

## 4. 排版系统
- 中文主字体：`"Noto Sans SC", sans-serif`
- 像素风标题/数字：`"VT323", monospace`
- 关键标题可选：`"Press Start 2P", monospace`（仅用于超短标签）
- 字号建议：
  - 正文：14px
  - 次要说明：11-12px
  - 模块标题：20-24px（VT323）
  - 大数字指标：28-52px（VT323）

## 5. 页面布局骨架
- 顶部导航：固定 `sticky`，高度约 56px。
- 主体三栏：`220px / 1fr / 260px`。
- 间距节奏：模块间距 12-16px，卡片内边距 12-16px。
- 内容层级：
  - 左栏：身份卡 + 菜单 + 周目标
  - 中栏：今日学习、课程主线、题库、错题、视频
  - 右栏：倒计时、预测、热力图、薄弱项、AI建议、统计、阶段路线

## 6. 组件规范

### 6.1 卡片（核心）
- 背景：`var(--bg-panel)` 或 `var(--bg-card)`。
- 边框：3px 实线，像素阴影必须保留。
- 圆角：默认不使用大圆角（保持像素硬边）。
- Hover：轻微上移 `translate(-1px, -1px)` + 强化边框色。

### 6.2 导航按钮
- 默认：低对比文本 + 透明背景。
- 激活：高亮边框 + 模块底色 + 发光文本阴影。
- 状态必须包含：`default / hover / active`。

### 6.3 任务项（Task Item）
- 左侧可点击勾选框。
- 已完成态：勾选框填充 + 文本删除线。
- 标签色区分类型：讲义/练习/视频/真题/复盘。

### 6.4 单元卡（Unit Card）
- 顶部 3px 彩色条表示单元类型或阶段。
- 包含：单元编号、标题、描述、进度条、进度文本。
- 锁定态：降低透明度 + 禁用光标。

### 6.5 右栏追踪组件
- 倒计时：大数字 + 考试目标信息。
- 热力图：7列网格 + 强弱分级颜色。
- 薄弱项：横向短条 + 文本标签 + 等级文字。
- AI 建议：紫色边框信息框，保持“提示卡”语义。

## 7. 动效规范
- 动效总体要轻：`0.1s ~ 0.2s`，避免花哨。
- 保留以下轻动效：
  - 进度条呼吸
  - 指标闪烁（低频）
  - hover 上移
- 不要使用大范围位移或复杂 3D 动画。

## 8. 响应式要求
- `<=1200px`：三栏降级为双栏。
- `<=900px`：改为单栏，右侧追踪模块下移。
- 顶部导航允许横向滚动，按钮不换行。
- 触控设备上，点击区域最小 40px。

## 9. 可访问性与可读性
- 正文对比度不低于 WCAG AA（建议 >= 4.5:1）。
- 颜色不作为唯一信息来源（进度同时显示数字/文本）。
- `:focus-visible` 必须有明显边框反馈。
- 避免纯亮黄背景直铺，需叠加轻微纹理/扫描线降低刺眼感。
