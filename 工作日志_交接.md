## 3. 本轮重点修复（高优先）

## 3.1 云同步与账号隔离（已实现于 `supabase.js`）

### A. 根因（修复前）
- 所有业务数据都写在全局 localStorage key（如 `fq-tasks-v3`），未做账号隔离。
- 登录后自动拉取逻辑在“云端为空”时存在“回推本机”行为，可能把本机旧数据污染到新账号云端。
- 手动“云端拉取覆盖本机”缺乏全链路超时/阶段状态，容易卡在 syncing 文案。

### B. 已落地修复
1. **工作空间隔离层**
- 新增：
  - `fq-workspace-guest-v1`
  - `fq-workspace-user-<uid>`
  - `fq-active-workspace-v1`
  - `fq-workspace-migrated-v1`
- 在 `supabase.js` 内安装 localStorage bridge：
  - 业务 key 读写自动路由到当前工作空间（guest 或 user:uid）。
  - app.js 无需改接口即可获得隔离能力。
- 首次迁移：旧全局业务数据迁入 guest 工作空间并清理旧 key。

2. **认证切换流程重构**
- onAuthStateChange 时顺序改为：
  1) 切工作空间（登录到 user:uid，登出到 guest）
  2) 刷新 UI
  3) 执行登录后同步决策
- 效果：
  - 未登录只看 guest 数据。
  - 不同账号只看各自空间，不再串号。

3. **登录后同步决策弹窗**
- 检测到“云端有数据 + 本机有数据”时弹窗三选一：
  - 使用云端覆盖本机（推荐）
  - 本机上传到云端
  - 稍后处理
- 若云端为空：默认不回推、不自动上传，提示“可手动上传”。

4. **手动拉取卡死修复**
- `runManualPullFromCloud` 增加 25s 总超时 watchdog。
- `pullAllFromCloud` 分阶段状态：正在读取任务/错题/学习日志/设置。
- 各查询增加 12s 超时。
- finally 必定释放 `syncActionPending`，按钮恢复可点击。

5. **删除危险回推逻辑**
- 已移除“云端为空时保留本地并回推云端”的行为，避免跨账号污染。

6. **tasks 云字段兼容扩展**
- 上传：支持 `planned_date / working_date / completed_at / rollover_count / rollover_mode`。
- 拉取：兼容读取上述字段并映射回本地对象。

## 5. 已踩坑与经验总结
1. **入口混淆坑**
- 根目录 `index.html` 才是运行页面；`app/index.html` 不是。
- README 曾提到 `site/` 路径，易导致误开旧路径。

2. **“改了但没生效”排查顺序**
- 先确认打开页面路径。
- 再确认 script 引入的是当前 `app.js/supabase.js`。
- 最后确认浏览器强刷（Cmd+Shift+R）与缓存问题。

3. **同步流程坑**
- 任何“自动回推”在多账号共用设备场景都非常危险。
- manual pull / upload 必须有 timeout + finally 清锁。

4. **Git 操作注意**
- 工作区可能有无关未跟踪文件（如 `制作规范_v1.0.md`），提交时务必显式挑选文件，避免误提交。
